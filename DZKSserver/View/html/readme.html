/

Тип запроса Ссылка Объяснение
GET / - домашний репозиторий. Выводит надпись server is working.
GET /wallpaper - возвращает картинку. Нужно для функционала виджета.
GET /readme - для просмотра этой справки.
GET /d - получение шаблона модели письма.
GET /userTemplate - получение шаблона модели пользователя.

/letters

Тип запроса Ссылка Объяснение
GET /letters - получение всех писем из БД.
GET /letters/{id:int} - возвращает картинку. Нужно для функционала виджета.
POST /letters - создание письма и внесение его в БД.
В теле запроса должна быть передана модель письма
POST /letters/user:{userId:int} - для добавления письма определенным пользователем.
В теле запроса должна быть передана модель письма, с измененными параметрами.
Для просмотра шаблона модели письма, нужно отправить запрос на /d.
PUT /letters/id:{id:int}/status:{status:bool}/inspect:{inspect:bool} - изменение статуса и inspect письма по его id. Все нужные параметры получаются из заголовка.
DELETE /letters/{id:int} - для удаления письма по его id, который получается из заголовка.

/users

<
Тип запроса Ссылка Объяснение
GET /users - получение списка всех пользователей на БД.
GET /users/id:{userId:int} - получение пользователя по его id.
GET /users/id:{userId:int}/letters - получение списка писем пользователя по его id.
POST /users/registration - для создания пользователя в БД.
Для этого нужно в теле запроса отправить модель пользователя.
Чтобы получить шаблон модели пользователя нужно сделать GET запрос на /userTemplate.
POST /users/login - для проверки правильности введеных данных.
В теле запроса нужно передать следующую конструкцию:
{
"login": "login",
"password": "password"
}

/archive

GET /archive - получение писем из архива.
GET /archive/{id:int} - получение письма по id из архива.
DELETE /archive/{id:int} - удаления письма по id из архива.
